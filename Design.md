## コンセプト

外国人児童向けの国語補助教材アプリ（GAS + React、iPad想定）。「教える」から「見せる」へ ── 文化背景の違いによる「言葉のスキマ」を、1枚の写真・イラスト・短い動画で直感的に補うという教材コンセプトに基づき、すべてのビジュアルは**大きく・丸く・力強く**、共有iPadでの誤操作を防ぐ設計を貫く。

- 対象: 小学校 外国人児童（美濃加茂市教育委員会）
- 実装: Google Apps Script + React 18 (CDN) + Tailwind
- 参照元: `index.html` / `コード.js`（プロジェクト添付コード）

### 学年パレット（oklch生成、本のモチーフ専用）

学年ごとに識別しやすい oklch の虹色パレット（`GRADE_PALETTE`、例: 1冊目 `oklch(66% .18 25)`、2冊目 `oklch(82% .19 96)`、3冊目 `oklch(66% .15 150)`、4冊目 `oklch(60% .13 250)`…と学年数ぶん循環）。各色は明るい `base` と濃い `dark` の2段階で構成し、**学年選択の本の背表紙**（`linear-gradient(100deg, base 60%→white 0%, base 16%, base 78%, dark 100%)`）と、**単元選択の見開き本の背表紙・ページ端・付箋のテープ**の両方で共通使用する。

このパレットは「本」モチーフ専用の例外であり、ボタンやナビゲーション（戻る等）には使わない。

## タイポグラフィ

- **見出し書体（Klee One / 丸ゴシック優先）**: `.round-font`クラス（`Arial Rounded MT Bold` → `Hiragino Maru Gothic ProN` → `Klee One` の順）。本のモチーフ内のテキスト（学年数字・「ねんせい」・単元名の付箋・単語画面の単元見出し）専用。単語カードの見出し・ふりがな（`.word-label`）はKlee One優先の別スタックで、手書き風。ルビ（ふりがな）は漢字部分にだけ自動で振られる独自アルゴリズムを使用。
- **UI書体（sans-serif）**: それ以外全て。太いウェイト（`font-black` / `font-extrabold`）を多用し、子どもにもはっきり読める設計。
  - 見出し: 900 / 36px
  - 単元見出し: 900 / 24px
  - 本文（意味説明）: 800 / 18px
  - サブ・補助テキスト: 700 / 14px
  - 注記・免責文: 600 / 11px
- **可変長テキストの縮小フォールバック**: 単元カードの単元名など、スプレッドシート由来で長さが不定なテキストは、文字数に応じてフォントサイズを段階的に縮小し（例: 21px→17px→14px）、それでも収まらない場合は `text-overflow: ellipsis` で1行に収める。折り返しによるカードの高さ崩れを避けるため。

## 形・余白・影

カード・ボタンともに大きな角丸を採用し、iPadでタップする対象であることを視覚的に示す。

- ボタン: 基本的に `rounded-full`（丸ボタン/ピル型）
- カード・モーダル: 2〜3rem の大きな角丸（`0.75rem` / `1rem` / `1.5rem`＝カード / `2rem`＝単語カード / `full`＝ボタン の5段階）
- 本・付箋モチーフのカード（学年の背表紙・単元の付箋）だけは例外的に角丸をごく小さく（`3px`〜`4px`）し、紙・本らしい質感を優先する。

| シャドウ | 用途 |
|---|---|
| `shadow-md` | 単元カード・小コンポーネント |
| `shadow-lg` | 単語カード・主要ボタン |
| `shadow-2xl` | モーダル（もくじ・注意事項） |

## 背景システム

画面の性質によって背景の作り方を2パターンに使い分ける。

1. **固定サイズの舞台（学年選択・単元選択）**: スクロールしない一枚絵の画面。デザイン上の固定キャンバス（`DESIGN_W`×`DESIGN_H` = 860×620）を基準に、`ResizeObserver` で計測した実際のビューポートに対して `contain`（`Math.min(w/DESIGN_W, h/DESIGN_H)`）でCSS `transform: scale()` し、常に中央配置する。**背景（グラデーション＋本棚シルエット）は必ずこの拡大縮小される舞台の「内側」に1枚だけ描画する**こと。舞台の外側（余白）に等倍の背景レイヤーを別途重ねると、拡大率の違いで本の密度が不連続になり継ぎ目が見えてしまう（過去に発生した不具合）。iPadの画面比率（横長 約1.33〜1.52）を想定し、`contain`により要素が画面外に切れることはないが、比率によっては上下または左右にわずかな余白（グラデーションのみ）が出ることを許容する。
2. **スクロールするコンテンツ（単語表示・この教材について）**: `position: fixed; inset: 0; z-index: -1` の背景レイヤーをコンテンツの下に敷く。**この2画面は本棚シルエットを使わず、グラデーションのみ**（コンテンツ量が多く、模様があると賑やかになりすぎるため）。

背景グラデーション自体はどちらのパターンも共通: `linear-gradient(120deg, oklch(96% .03 300) 0%, oklch(96% .03 160) 35%, oklch(96% .04 60) 70%, oklch(96% .03 30) 100%)`。

## コンポーネント

- **ボタン**: 主要CTA（例: 「▦ もくじ」）はインディゴ背景＋白文字＋強い影。押下時は `active:scale-95` でわずかに縮むフィードバック。角丸は用途を問わず基本 full。
- **戻るボタン（BackButton）**: 全画面共通でslate-100の円形＋slate-700アイコン。スクロールしない画面（学年選択・単元選択）では `position: fixed` で画面左上に固定表示、スクロールする画面（単語表示・この教材について）ではsticky headerの先頭に配置。ヘッダーバーにタイトル文言は置かない（ボタンのみ）。
- **単語カード（WordCard）**: 上部に写真/イラスト/動画（16:9枠）、左上にKlee Oneのふりがな付きラベル、下部に意味説明文。横長画像は背景ぼかし＋前面containの二重表示。複数画像は左右の丸い矢印ボタンで送る。動画はGoogle Driveのiframeプレビュー。
- **単語表示画面の単元見出し**: 単元選択画面の付箋カードと同じ見た目（クリーム背景＋上部に学年カラーのテープ＋丸ゴシック体）で統一。以前のインディゴ左ボーダー見出しは廃止。
- **学年スパイン（本棚の本・GradeSpine）**: oklchグラデーションの背表紙。タップで press → cover（浮き上がり）→ opening（ページがめくれる）の3段階アニメーションで単元選択へ遷移。
- **単元スパイン（見開き本・UnitSpread）**: 学年選択の延長として、単元選択も「本」のモチーフで統一。見開いた本（スパイン・ページ端の重なり・罫線を描画）の左右ページに、単元名を書いた付箋風カード（テープ留め・わずかな回転・紙のような小さい角丸）を最大4件ずつ、計8件まで配置。9件以上ある学年は8件ごとにページ分割し、左右の矢印ボタンとページドットで送る。単元名はスプレッドシートの実データ（`unitId`/`unitRuby`）を`Ruby`コンポーネントでふりがな表示。カードタップで press → 白フェード（学年選択と同じ8分割ストライプ演出を再利用）→ 単語一覧へ遷移。
- **スティッキーヘッダー**: 半透明白＋ぼかし（`backdrop-filter: blur`）、下部に細い境界線。タイトル文言は置かず、戻る・もくじ等のボタンのみを配置する。
- **モーダル（もくじ / 注意事項）**: 全画面をスレート900の80%透明＋ぼかしで覆い、モーダルは中央配置。閉じるボタンは常に右上の丸いXボタン。
- **セクション見出し**: 太い左ボーダー（`border-left: 8px solid indigo-500`）による強調（本文セクション内の小見出し用途。画面全体の見出しには使わない）。

## モーション・操作性

| 演出 | 内容 |
|---|---|
| 押下 (130ms) | `scale(0.94)` — ease |
| 浮き上がり (380ms) | `scale(1.22) translateY(-10px)` — `cubic-bezier(.25,.8,.3,1)` |
| 本が開く／めくれる (480〜620ms) | 8分割ストライプが遅延付き `rotateY` で開き、白フェードで画面遷移。学年選択（480ms）・単元選択の付箋タップ（130ms押下 + 620msフェード）で同じ仕組み（`screenStripStyle` / `gradeOverlayStyle`）を共有 |

通常のタップ操作はすべて `active:scale-95` の即時フィードバックのみ。学年選択と単元選択の2画面だけが「本を開く／めくる」特別な演出を持つ、儀式的な導線（他の画面には広げない）。

**押下時に背景色が消える不具合への対処**: グローバルCSSに `button:focus, button:active { background-color: transparent !important; }` という誤操作防止ルールがあり、これがボタン固有の背景色（インラインstyleやTailwindクラス）を`!important`で上書きしてしまう場合がある（例: 白背景のボタンが押下時に透明になり背景が透けて見える）。対策として、色を保ちたいボタンには専用クラス（`.about-btn` / `.mokuji-btn` / `.back-btn`）を付け、`button.xxx-btn:focus, button.xxx-btn:active { background-color: 元の色 !important; }` で個別に打ち消す。新しいボタンで同様の現象が出た場合はこのパターンに倣うこと。

**共有iPad向けの操作制約**:
- ページ全体で text-select / タップハイライト / フォーカスリングを無効化（誤操作防止）
- viewport は `user-scalable=no` で固定、ピンチズーム不可
- 画像読み込み失敗やデータ取得エラー時は、赤系の専用エラー画面と「再読み込み」ボタンを表示

## コンテンツ・トーン

UI文言はひらがな中心で短く（「もくじ」「ねんせい」「じゅんび中」）、児童にもそのまま読める語を選ぶ。一方「この教材について」ページは大人（教員）向けの説明文で、収録／非収録の判断基準を丁寧な文体で明示する、というトーンの使い分けがある。

収録しない語彙の判断基準: ① 動作化で伝わるもの　② 教科書と価値が重複するもの　③ 写真入手が困難でイラスト化も理解を助けないもの　④ 中立性の観点からイラスト化を避けるべきもの

素材は生成AI（Google Gemini 等）で制作している旨を、学年選択画面下部に常時明記。教材の公開範囲・二次利用禁止についての注意事項は、以前は常時表示のテキストだったが、現在はリンクをタップした時だけモーダルで表示する形に変更している。

## 新規画面実装時のガイドライン

1. 背景・本文・境界線は必ずスレート系、主要アクションのみインディゴで強調する（学年選択・単元選択の2画面のみ、oklch虹色パレット＋本のモチーフの例外）。
2. ボタンは `rounded-full`、カード・モーダルは 1.5〜2rem 台の大きな角丸を基本とする（本・付箋モチーフのカードのみ例外的に角丸を小さくする）。
3. 単語ラベル・ふりがな表示以外のUIテキストにKlee Oneを使わない（Klee Oneは単語カード専用）。本のモチーフ内のテキストは `.round-font`（丸ゴシック優先）を使う。
4. 通常のインタラクションは `active:scale-95` のみに留め、独自の複雑なアニメーション（本を開く／めくる演出）は学年選択・単元選択のような特別な意味を持つ場合に限定する。安易に他の画面へ拡張しない。
5. 共有iPadでの誤操作防止（text-select無効化・ピンチズーム不可・フォーカスリング無効化）を新規画面でも踏襲する。
6. スクロールしない一枚絵の画面を追加する場合は「背景システム」章の固定キャンバス＋`contain`スケール方式を、スクロールするコンテンツ画面を追加する場合はグラデーションのみの固定背景を踏襲する。
7. 戻るボタンは学年カラー等に連動させず、必ずニュートラル配色（slate-100 / slate-700）で統一する。
8. ボタンに独自の背景色を持たせる場合、押下時に色が消えないか（上記「押下時に背景色が消える不具合」）を確認する。

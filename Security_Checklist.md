# セキュリティチェックリスト(Jzukan / にほんごずかん)

このプロジェクト(Google Apps Script + HtmlService によるWebアプリ、React 18 + Babel Standalone + Tailwind Play CDN をノービルドでCDN読み込みするフロントエンド、Google スプレッドシートをデータソース、Google Drive上の画像/動画を埋め込み)でコーディング・`clasp push`/デプロイする際に、セキュリティ上の問題を作り込まないためのチェックリスト。

- 新機能の実装・`clasp push`/デプロイ前後のセルフチェックに使う
- 該当しない項目は理由とともにスキップしてよい
- 各セクション先頭の優先度目安(🔴高 / 🟡中 / ⚪低)。時間が無いときは🔴から確認する

## 0. このアプリの脅威モデル(前提)

- **公開範囲:** `appsscript.json` の `webapp.access` が `ANYONE_ANONYMOUS` — ログイン不要で誰でもアクセスできるWebアプリ。`executeAs: USER_DEPLOYING` のため、サーバー側コード(`コード.js`)はデプロイしたGoogleアカウントの権限で実行される
- **資産:** ①バインドされたGoogleスプレッドシートの内容(現状は学習コンテンツで機密性は低い想定だが、シート自体はGoogle側で非公開のまま、アプリ経由でのみ一部を公開する設計)②Google Driveにホストされた画像/動画ファイル ③現状 npm 依存なし、APIキーなし、ユーザーアカウント・ログイン・個人情報保存機能なし
- **主な脅威:** 「認証突破」よりも「①スプレッドシートの意図しない列/データがそのままクライアントに漏れる」「②ビルドレスでCDN(unpkg / Tailwind Play CDN)から実行時に読み込むJSが改ざん・差し替えされた場合に、公開Webアプリの全訪問者に影響する(サプライチェーン)」「③エラーメッセージ経由の内部情報漏洩」に寄っている
- 将来ユーザー投稿機能・認証機能・課金APIなどを追加する場合、このチェックリストは前提が変わるため大幅な見直しが必要(その際はTatoeba-appの `Security_Checklist.md` の構成も参考になる)

---

## 1. Webアプリ公開範囲・認可 🔴

- [ ] `appsscript.json` の `webapp.access` (`ANYONE_ANONYMOUS` / `ANYONE` / `DOMAIN` 等)が、意図した公開範囲になっているか変更のたびに確認している
- [ ] `webapp.executeAs` を変更する場合、実行アカウントの権限(バインドされたシート・Drive等へのアクセス範囲)を理解した上で変更している
- [ ] デプロイ(`clasp deploy`)を新規作成・更新する際、Apps Scriptエディタ側の「デプロイ管理」で公開設定が意図通りか確認している(`.clasp.json`/`appsscript.json` の設定と、実際のデプロイ設定がずれることがある)

## 2. データ露出範囲(スプレッドシート → クライアント) 🔴

`コード.js` の `getSheetData()` は、シート名に「年生」を含む全シートの全行・全列を無条件に `obj[header] = row[i]` で詰めてクライアントに返す設計になりやすい。

- [ ] `getSheetData()` (または同様のデータ取得関数)が返すフィールドは、フロントエンドが実際に使用するキーのみに絞っている(全列をそのまま返さない)。コンテンツ編集者がスプレッドシートに内部用メモ・下書き列・正答以外の情報を追加しても、それが自動的に公開されない設計にする
- [ ] 新しい列をスプレッドシートに追加した場合、それが自動的に公開APIの戻り値に含まれてしまわないか確認した(明示的なフィールド一覧への追加が必要な設計を維持する)
- [ ] スプレッドシート自体の共有設定(「リンクを知っている全員が閲覧可」等になっていないか)を、Webアプリ経由の公開とは別に確認している

## 3. フロントエンド依存関係・サプライチェーン(CDN読み込み) 🔴

このプロジェクトはビルドステップ・`package.json`・ロックファイルを持たず、`index.html` の `<script>` タグで実行時にCDNから直接JSを読み込んでいる(React / ReactDOM / Babel Standalone / Tailwind Play CDN)。npmの脆弱性スキャンが効かない代わりに、**CDN配信元の改ざん・差し替えがそのまま全訪問者への攻撃になる**リスクがこのプロジェクトの主な「依存関係リスク」。

- [ ] React / ReactDOM / Babel など、**静的な配信元(unpkgの特定バージョンURL等)を使うスクリプト**は、`@18` のような浮動バージョン指定ではなく `@18.3.1` のように**厳密なバージョンに固定**している
- [ ] 上記の固定バージョンスクリプトには **Subresource Integrity (`integrity` 属性 + `crossorigin="anonymous"`)** を付与し、配信元が差し替えられても改ざんされたコードが実行されないようにしている
- [ ] `https://cdn.tailwindcss.com` (Tailwind Play CDN) は**本番運用非推奨**であることを認識している(Tailwind公式が明言。リクエストごとに内容が変わり得るため原理的にSRIを付与できない)。当面CDN読み込みを続ける場合はこのリスクを受容していることを明示し、将来的にはビルド済みCSS(Tailwind CLI等)への移行を検討する
- [ ] 新しく外部CDNスクリプト/フォント/スタイルシートを追加する場合、信頼できる配信元か、SRIを付与できるか(バージョン固定された静的ファイルか)を確認している
- [ ] Babelバージョンを変更する場合、挙動差分(JSXトランスパイル結果の変化)を確認している。現状 `7.12.9` に固定している理由(import/export命令のバグ回避)がコメントに残っているため、アップグレード時はその回避策が引き続き必要か確認する

## 4. 外部埋め込みコンテンツ(Google Drive) 🟡

- [ ] `drive.google.com/thumbnail?id=...` / `drive.google.com/file/d/.../preview` に渡す Drive ファイルIDは、スプレッドシート由来の値であることを認識し、**想定外のドメインへ渡らない**設計(`drive.google.com` 固定のテンプレートリテラルで組み立てる、任意URLをそのまま埋め込まない)を維持している
- [ ] `<iframe>` で外部コンテンツ(Drive動画プレビュー等)を埋め込む場合、`allow` 属性は必要最小限の権限(現状 `autoplay` のみ)に絞っている
- [ ] Drive上のファイルの共有設定(「リンクを知っている全員が閲覧可」)が、埋め込み表示に必要な範囲を超えて公開されていないか確認している

## 5. エラーハンドリング・情報漏洩 🟡

- [ ] `google.script.run` の `withFailureHandler` でクライアントにそのまま表示している `err.message` に、Apps Script側の内部実装詳細(関数名、スタック情報等)が含まれすぎていないか確認している。学習アプリの利用者(子ども・保護者・教員等)に見せて問題ない一般的なメッセージに留める
- [ ] `appsscript.json` の `exceptionLogging: STACKDRIVER` により送られるログに、個人情報や機密情報が含まれていないか確認している

## 6. シークレット・認証情報管理(将来のAPI追加に備えて) 🔴

現状APIキー等の秘匿情報は無いが、今後 翻訳/画像生成/音声合成 等の外部有料APIを追加する場合に備えて:

- [ ] APIキー等を `コード.js` に直書きせず、`PropertiesService`(Script Properties)経由で読み込む設計にする
- [ ] 有料外部APIを追加する場合、クライアントから送られた入力をそのまま上流に転送せず、必要なフィールドのみサーバー側(Apps Script側)で組み立てる。呼び出し回数の上限・レート制限の要否も検討する(Tatoeba-appの `Security_Checklist.md` §2 が詳しい実例)
- [ ] `console.log`(Apps Scriptの `Logger.log`/`console.log`)の出力にAPIキー・個人情報が混入していないか確認する

## 7. クライアント側実装(一般原則) ⚪

- [ ] `dangerouslySetInnerHTML` や `innerHTML` を使っていない、または使う場合はサニタイズしている(現状未使用)
- [ ] 外部から取得した文字列(スプレッドシートの内容等)を `eval` / `new Function` に渡していない
- [ ] `localStorage`/`sessionStorage`/Cookie を新たに使い始める場合、保存する情報が公開端末(共有iPad等)で問題ないか検討する

## 8. デプロイ・clasp運用 🟡

- [ ] `clasp push` 前に `clasp status` で送信対象ファイルを確認し、意図しないファイル(ローカルの一時ファイル、デバッグ用コード等)が含まれていないか確認している
- [ ] `clasp login` に使っているGoogleアカウントの権限範囲を把握している(このアカウントの権限で `executeAs: USER_DEPLOYING` が実行されるため)
- [ ] `.clasp.json` の `scriptId` は機密情報ではないが、意図せず無関係な場所へ公開UIとして共有しないよう扱う

## 9. push/デプロイ前の最終確認 🔴

- [ ] 変更内容に意図しないファイル(OS生成ファイル等)が含まれていないか確認した
- [ ] 新しい外部通信先(CDN、Drive以外の外部ドメインへのfetch等)が増えていないか、増えている場合は正当な理由があるか確認した
- [ ] `getSheetData()` 等データ取得関数を変更した場合、2章の「返すフィールドを絞る」方針を退行させていないか確認した
- [ ] `appsscript.json` の `webapp.access`/`executeAs` を変更した場合、1章の公開範囲チェックを再度行った
